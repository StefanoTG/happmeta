# SubProxy — Pasarguard Subscription Middleware

A production-ready FastAPI middleware that transparently proxies subscription URLs from a [Pasarguard](https://github.com/pasarguard) panel, lets you rewrite metadata headers and node names on the fly, and ships with a Telegram bot for live management.

```
Client ──► https://sub.mydomain.com/sub/abc123
              │
              ▼
        SubProxy (FastAPI + nginx)
              │  fetch
              ▼
        https://panel.real:8443/sub/abc123
              │  rewrite headers + node names
              ▼
        Modified subscription returned to client
```

The client never sees the real panel domain. Pasarguard's built-in **URL Prefix** field handles link generation — no manual URL crafting required.

## Features

- **Transparent reverse proxy** for any Pasarguard subscription path
- **Header rewrites**: inject, override, or remove `profile-title`, `profile-web-page-url`, `support-url`, `subscription-userinfo`, `content-disposition`, etc. Disabled headers are also stripped from the upstream response.
- **Node renaming**: prefix / suffix / emoji / regex / template rules
- **Format-aware**: handles Base64 V2Ray bundles, Clash/Mihomo YAML, sing-box JSON, plus raw VLESS/VMess/Trojan/Shadowsocks links
- **Telegram bot** with inline-keyboard UI to manage everything live
- **SQLite** storage — no hardcoded values
- **Rate limiting**, structured logging, timeouts, path validation
- **One-line installer** with nginx, systemd, Certbot, and venv set up automatically

## One-line installation

```bash
curl -fsSL https://raw.githubusercontent.com/StefanoTG/happmeta/main/install.sh | sudo bash
```

The installer prompts for:

| Question | Example |
|---|---|
| Telegram bot token | `123:ABC` |
| Telegram admin numeric ID | `123456789` |
| Pasarguard panel host | `panel.example.com` |
| Pasarguard port | `8443` |
| Pasarguard scheme | `https` |
| Public middleware domain | `sub.mydomain.com` |
| Optional node prefix/suffix | `🔥 ` / ` | Premium` |
| Configure HTTPS via Certbot? | `y/n` |

When it finishes, set Pasarguard's **URL Prefix** to `https://sub.mydomain.com` and you are done.

## Telegram bot

Send `/start` to your bot. The menu offers:

- **📋 Metadata** — list / add / edit / toggle / delete header rules
- **🏷 Node Rules** — list / add / toggle / delete rename rules (prefix, suffix, emoji, regex, template)
- **⚙️ Current Config** — show effective config and counts
- **🔄 Reload Config** — re-read `config.json` in bot process
- **♻️ Restart Proxy** — `systemctl restart subproxy-api` (via sudoers drop-in)

## Header override logic

For every metadata row in the SQLite `metadata` table:

| Row state | Effect on response |
|---|---|
| `enabled = 1` | Value **replaces** upstream's header of the same name (or is injected if absent). |
| `enabled = 0` | Header is **removed** entirely, even if Pasarguard sent it. |

Headers not present in the table are passed through unchanged. Hop-by-hop headers (`transfer-encoding`, `content-length`, `content-encoding`, …) are always stripped because we may rewrite the body.

## Node renaming

Rules are evaluated in `priority` order (ascending). Each rule type:

| Type | Pattern | Replacement | Result |
|---|---|---|---|
| `prefix` | — | `🔥 ` | prepended |
| `suffix` | — | ` | Premium` | appended |
| `emoji` | — | `🔥` | injected at start if missing |
| `regex` | `^DE` | `🇩🇪` | `re.sub(pattern, replacement, name)` |
| `template` | `{name} ❄` | — | template with `{name}` placeholder |

Rules are applied across all supported formats consistently.

## Service management

```bash
systemctl status  subproxy-api subproxy-bot
systemctl restart subproxy-api subproxy-bot
journalctl -u subproxy-api -f
journalctl -u subproxy-bot -f
```

Logs are mirrored to `/opt/subproxy/logs/subproxy.log` (rotated, 5 MB × 3).

## Manual / development run

```bash
cd /opt/subproxy
source venv/bin/activate
SUBPROXY_CONFIG=$PWD/config/config.json python main.py     # FastAPI
SUBPROXY_CONFIG=$PWD/config/config.json python run_bot.py  # bot
```

## Security notes

- Configuration file is created with `chmod 600`.
- `slowapi` enforces a per-IP rate limit (default 120/min, configurable).
- Path traversal (`..`) and absolute paths are rejected before forwarding.
- The sudoers drop-in only allows `systemctl restart subproxy-api`, nothing else.
- Always run behind HTTPS in production (the installer can set up Certbot for you).
